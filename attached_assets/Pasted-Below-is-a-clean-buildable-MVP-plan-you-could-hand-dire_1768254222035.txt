1. MVP — One-Sentence Definition

An AI-powered investing copilot that lets individual investors ask a question (text or voice) and instantly receive a synthesized investment thesis with probabilities, charts, and actionable scenarios—without needing multiple tools.

2. MVP Scope (Very Important)
MVP INCLUDES

✅ Natural language prompt (text first, voice optional later)
✅ Single query → structured investment output
✅ Scenario analysis (Base / Bull / Bear)
✅ Assigned probabilities (clearly labeled as estimates)
✅ Simple charts (price history + projected ranges)
✅ Source transparency (what data was used)
✅ Save the thesis (local/session-based)

MVP EXCLUDES

❌ Auto-trading / brokerage execution
❌ Real-money actions
❌ Personalized fiduciary advice
❌ Guaranteed predictions
❌ Paid/proprietary data (Phase 2)

3. The 3 MVP Screens

This maps perfectly to what you described.

Screen 1 — Ask / Explore (The Hook)

Purpose: Let the user “vibe code” their investment question.

UI Elements

Large input box (text)

Example prompts:

“How could I profit from instability in Venezuela?”

“Is Chevron a good 3–5 year investment?”

“Short-term vs long-term oil plays?”

Optional 🎤 Voice input (Phase 1.5)

“Analyze” button

Why this works

Zero friction

Matches your “talk it in / text it in” vision

Immediately replaces 5–10 tools

Screen 2 — Thesis & Scenarios (Core Value)

Purpose: Deliver the synthesized intelligence.

Sections

Plain-English Thesis

5–8 bullets

What’s happening

Why it matters

What could go wrong

Scenarios with Probabilities

Bull Case (30%)

Base Case (45%)

Bear Case (25%)
(probabilities must sum to 100%)

Key Metrics

Beta

Volatility

Market cap

Float (if applicable)

Risk notes (manipulation, liquidity)

Timeline

Short-term (0–6 months)

Medium (6–24 months)

Long-term (2–5 years)

Why this works

Advisor-like output

Clear, decisive

Avoids analysis paralysis

Screen 3 — Charts & Credibility

Purpose: Build conviction and trust.

Charts

Price history (12–60 months)

Projected range bands (shaded)

Volatility overlay

Credibility Panel

Data sources used (Yahoo Finance, news APIs, etc.)

Confidence level (High / Medium / Low)

Disclosure banner:

“Probabilistic analysis. Not financial advice.”

Optional (still MVP-safe)

“Save Thesis”

“Set Reminder” (fake button / coming soon)

4. End-to-End User Flow

  ↓

  ↓
AI analyzes → pulls data → synthesizes
  ↓
Thesis + scenarios + probabilities
  ↓

No accounts required for MVP.

5. AI / Agent Architecture (Simple but Powerful)

You do not need complex agent orchestration yet.

MVP Agent Roles (Behind the Scenes)

Agent 1 — Research Agent

Pulls public data:

Price history

News summaries

Macro context

Agent 2 — Synthesis Agent

Builds thesis

Creates scenarios

Assigns probabilities

Flags uncertainty

Agent 3 — Presentation Agent

Formats output

Produces chart-ready data

Generates UI-friendly JSON

All of this can be a single GPT call with structured prompting in MVP.

6. Data Sources (MVP-Friendly)

Public news APIs or scraped summaries

General macro knowledge (LLM-native)

Phase 2

Paid financial APIs

Investor performance data
