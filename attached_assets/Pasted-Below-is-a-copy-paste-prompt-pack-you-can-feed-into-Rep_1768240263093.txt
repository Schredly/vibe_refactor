Below is a copy/paste prompt pack you can feed into Replit to build an “elegant wizard” app that:

loads a Google Drive doc (or uploaded file),

extracts questions into a clean UI,

lets you Record an answer per question (speech-to-text),

collapses answers under each question,

produces a GPT summary of the MVP understanding,

then generates collapsible prompt bundles for building the MVP,

and finally hands those prompts into Replit’s “create/generate app” flow (as much as Replit allows from inside an app).

I’m writing these prompts assuming a modern Replit stack (Next.js or React + Node). If Replit chooses a different stack, it should still follow the same behavior.

## Prompt 2 — Implement speech-to-text recording per question (clean + reliable)

**Prompt to paste into Replit:**

Enhance MVP Wizard’s “Capture Answers” step with a reliable recording + transcription system.

### Requirements
- Default transcription: **Web Speech API** (SpeechRecognition / webkitSpeechRecognition)
- Provide a fallback mode: manual typing only.
- Make the transcription per-question, not global.

### UX details
- When user clicks Record on a question:
  - show mic “recording” indicator on that card only
  - start streaming interim transcript into the collapsible Answer area
- When user clicks Stop:
  - finalize transcript, store in question.answerText
- Re-record:
  - confirm “Replace existing answer?” and then overwrite
- Edit text:
  - allow manual editing of answerText in a textarea
- Collapsible Answer area:
  - default collapsed if answer exists, expanded when recording or empty

### Robustness
- Handle permission denied
- Handle mobile / unsupported browsers
- Auto-restart recognition if it stops unexpectedly while recording
- Debounce transcript updates to avoid UI jitter
- Store partial transcripts so nothing is lost

### Optional audio capture
If easy: also record raw audio via MediaRecorder and store it as a blob reference, but this is optional. The key is accurate text capture.

### Developer architecture
Create a reusable hook like:
- useSpeechTranscription({ onFinal, onInterim, lang })

Make sure the system supports different languages (default en-US).

## Prompt 4 — Generate “Build Pack” prompts (no detailed data model)

**Prompt to paste into Replit:**

Implement “Generate Build Pack” so it creates prompt bundles that can be pasted into Replit Agent to generate an MVP.

### Rules
- Do NOT produce a detailed data model (no exhaustive schemas).
- Keep storage requirements high-level: “store X fields” not full ERD.
- Focus on actions, screens, flows, and acceptance criteria.

### Output format
Return an array of prompt bundles:
Each bundle:
- category
- title
- content (markdown)
- collapsedByDefault true

Categories must include:
1) Product Overview (one-pager)
2) Primary User Flows (step-by-step)
3) UI Spec (screens + components)
4) Backend/API Spec (endpoints & responsibilities)
5) Storage (high-level only)
6) Edge Cases & Validation
7) Acceptance Criteria
8) V1 Cut List (what is explicitly out of scope)
9) “Replit Master Prompt” (single consolidated prompt)

### UI
- Show categories as accordion sections
- Each prompt has:
  - copy button
  - “edit” button
  - “regenerate this section” button
- Provide a “Copy Master Prompt” button at top

### Backend
Create `/api/generatePrompts` that uses the agreed summary + Q/A.
