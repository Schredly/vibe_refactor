MVP Screens (5)
Login & Role Selection Screen
Authenticate users and route them into the app with the correct role-based access (Technician, Manager, Admin).

UI Elements:

Email/username input field
Password input field
Login button
Error message area for invalid credentials
+1 more
This keeps access simple while enforcing role-based behavior from the very first step, which is essential for auditability and basic compliance.

Sample Intake Screen
Allow technicians to log new biomarker assay samples into the system quickly and accurately.

UI Elements:

Form fields for Sample ID, Patient/Subject ID, Assay Type, Collection Date/Time, and Priority
Dropdown for Initial Status (defaulted to 'Received')
Free-text Notes field for special handling instructions
Submit button to create the sample record
+1 more
A focused form with only essential fields enables rapid intake and reduces the chance technicians revert to spreadsheets because the system feels slow or cumbersome.

Sample Worklist & Status Screen
Provide technicians and managers with a live, sortable list of all samples and their current status, enabling status updates instead of offline tracking.

UI Elements:

Table listing samples with columns for Sample ID, Assay Type, Current Status, Assigned To (optional), Received Date, and Last Updated
Filter controls for Status, Date Range, and Assay Type
Search box for Sample ID or Subject ID
Inline or row-level 'Update Status' button that opens a small modal for changing status and adding a note
+1 more
A central, real-time worklist removes the need for shadow spreadsheets by giving all teams a shared, up-to-date view of sample progress and easy status updates.

Sample Detail & Audit Trail Screen
Show all key information for a sample, its status history, and approvals for regulatory traceability.

UI Elements:

Header with Sample ID, current status badge, and basic metadata (Assay Type, Priority, dates)
Read-only section for intake data (patient/subject ID, collection time, notes)
Status timeline or vertical list of events (status changes, notes) with user, timestamp, and action
Button for 'Approve Result' (visible to Managers only) with confirmation modal
+1 more
Consolidating sample data and its change history in one place satisfies audit and quality needs while allowing managers to confidently approve results within the same UI.

Manager Dashboard Screen
Give managers a live overview of sample volumes, statuses, and basic turnaround-time metrics without waiting for weekly reports.

UI Elements:

KPI tiles showing counts of samples by key statuses (Received, In Processing, Awaiting Approval, Approved)
Simple bar or donut chart visualizing current status distribution
Table of 'At-Risk' samples (e.g., oldest in processing) with received date and age in days/hours
Filters for date range and assay type to narrow the dashboard scope
+1 more
A concise, live dashboard addresses the explicit need to replace weekly reports and provides managers with actionable visibility to reduce turnaround time.

User Flow
Step 1: User navigates to the web app and logs in with their credentials.
Step 2: Technicians go to the Sample Intake screen and create new sample records as specimens arrive.
Step 3: All users view the Sample Worklist to see live status and filter/search for specific samples.
Step 4: Technicians open samples from the Worklist to update status and add processing notes as work progresses.
Step 5: When assay results are ready, Managers open the Sample Detail screen to review metadata, notes, and audit trail.
Step 6: Managers approve results using the 'Approve Result' action, which writes to the audit trail.
Step 7: Managers monitor throughput and bottlenecks using the Manager Dashboard with filters and KPIs.
Step 8: Managers export dashboard or worklist data to CSV for any additional analysis or reporting.
